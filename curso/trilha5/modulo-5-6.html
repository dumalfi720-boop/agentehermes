🧩 How to combine: real hybrid architecture

The right question is not “which one to choose”, it is “how to combine”. The everyday stack in 2026 mixes several tools — as long as you know where each one fits.

🏆 Recommended stack

The combination that covers 90% of cases:Hermes (proactive via msg, cron, gateway) + Claude Code (coding cockpit sitting on the PC) + GitHub (source of truth for context, skills, configs). All three point to the same repository.

🏗️ Why this stack wins

Hermes:tasks on the move, cron automations, gateway 20+ platforms, agent always on.
Claude Code:deep code reasoning, refactor, PR review, Claude Sonnet 4.5/Opus model.
GitHub:source of truth for CLAUDE.md, AGENTS.md, .hermes.md, skills/, soul.md. Guaranteed versioning.
Synergy:each covers different friction. Greater synergy than choosing just one.

📁 Typical core repo structure

repo/
├── README.md           # overview compartilhado

TOPIC 2

🔗 Point Claude Code and Hermes to the same repo

CLAUDE.md is in the root of the repo, read by Claude Code..hermes.md (or MEMORY.md/USER.md)It's in the same repo, read by Hermes. Each respects its own format without conflict.

📜 File convention

README.md:overview of the project that both agents read.
CLAUDE.md:specific instructions for Claude Code (long context, coding rules).
.hermes.md:specific instructions for Hermes (tone, preferences, reminders).
skills/:directory with SKILL.md — Hermes loads via progressive disclosure.
.cursorrules / .cursor/rules/*.mdc:Cursor rules that Hermes also reads.

⚠️Anti-pattern: single file for all

Trying to create a single file for all agents becomes a mess. Each agent has its own convention; respecting this simplifies maintenance. Common content goes in README.md; specific in the agent file.

TOPIC 3
3

🔁 n8n + Hermes

n8n runs deterministic visual workflow(synchronizes CRM, triggers webhook).Hermes adds adaptive intelligencewhere you need reasoning (summarize email, classify ticket, adjust tone).

🔄 Integration pattern

n8n receives trigger:Stripe webhook, GitHub event, schedule cron.
n8n decides if he needs LLM:payload-based IF condition.
n8n calls Hermes via HTTP:"HTTP Request" node points to the Hermes webhook (browser/http gateway).
Hermes reasons:summarize, classify, adjust tone. Returns JSON.
n8n continues the workflow:Use Hermes' answer for the next step (SendGrid, Slack, Postgres).

💡Cost Savings

n8n does the deterministic part without LLM (free once created). Hermes is called only where it returns value (adaptive reasoning). Cheap and powerful combo — you don't pay LLM on each workflow node.

TOPIC 4
4

🐍 LangChain inside Hermes

Hermes hasplugin system. You create a Python plugin that imports LangChain and exposes it as a tool. Hermes calls the tool when it needs complex RAG, document parsing or specialized retriever.

🧠 Usage pattern

Pluginlangchain-retriever:Python wrapper that connects to Pinecone/Weaviate/Chroma and exposes it as a Hermes tool.
Skillrag-search:Markdown procedure that guides the agent to use the plugin for retrieval.
Hermes reasons above:receives relevant excerpts from the retriever, synthesizes response, communicates via gateway.

🌐 Combination advantage

You get the LangChain ecosystem (500+ integrations, vector stores, document loaders) without having to build the agent from scratch. Hermes becomes the operational shell, LangChain the retrieval core.

TOPIC 5
5

🤖 Codex CLI or Aider as sub-agent

Skills bundledcodex e aiderallow Hermes to invoke Codex CLI or Aider as a subprocess for heavy coding tasks.Hermes coordinates, experts execute.

⚙️ Practical flow

You send Telegram:"refactor module X using Aider".
Hermes reads context:identifies repo, loads skillaider, mount appropriate prompt.
Hermes summons Aider: subprocesswith correct arguments. Aider performs refactor with optimized model.
Aider returns diff:Hermes receives, summarizes changes.
Hermes responds to you on Telegram:"Refactor ready. Changes to files X, Y. Diff saved indiffs/2026-05-10.diff."

💡Why delegate?

Aider has superior git integration (automatic commits, diff view). Codex has an optimized model for coding. Instead of Hermes doing average coding, he delegates it to a specialized tool. You get the best of each.

TOPIC 6
6

⚠️ Anti-pattern: force replacement

Trying to replace Claude Code with Hermes in heavy coding isfrustrating(model via Codex Plus is not Sonnet 4.5). Trying to replace Hermes with Claude Code in proactive automation too (no cron, no gateway).

✗ Common anti-patterns

  • ✗Hermes for heavy monorepo refactor
  • ✗Claude Code for marketing cron
  • ✗n8n for "respond in the company's tone"
  • ✗LangChain for personal agent MVP
  • ✗CrewAI for Telegram operational bot

✓ Patterns that work

  • ✓Hermes orchestra; Aider/Codex commit
  • ✓Claude Code reasons; Git is source of truth
  • ✓n8n integrates; Hermes reasons
  • ✓LangChain inside Hermes plugin
  • ✓CrewAI batch; Operational Hermes

📜Rule of thumb

Each agent does what he does best.Hermes orchestra. Claude Code reasons code. n8n integrates services. LangChain search in vector store. Aider commits clean diffs. Accepting division = more productive stack. Forcing outside the niche = extra work, low quality, daily frustration.
