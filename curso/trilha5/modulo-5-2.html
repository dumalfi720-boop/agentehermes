🟢Hermes vs OpenCode

OpenCode isPure open source coding CLI, focused on software development tasks. Hermes is a generalist with persistent message gateway, skills hub, MEMORY.md/USER.md. Hermes hasskill bundledopencodewhich delegates coding tasks to the OpenCode CLI — the two coexist.

🪽 Hermes wins in

  • ✓Full-life workflow (not just code)
  • ✓Automations via cron + gateway
  • ✓Communication via Telegram/WhatsApp
  • ✓Persistent memory between sessions
  • ✓Skills hub agentskills.io

⌨️ OpenCode wins in

  • ✓Pure software engineering
  • ✓Focus on code without distraction
  • ✓Terminal dev tasks
  • ✓OpenAI Compatible Multi-Provider
  • ✓Lightweight — only does coding

💡Coexistence

Skill bundledopencodeallows Hermes to invoke OpenCode as a subprocess. You say "Hermes, use OpenCode to refactor src/auth/" and Hermes coordinates, OpenCode executes.

🤖 Hermes vs OpenAI Codex CLI

Codex CLI is OpenAI's official CLI, requires OpenAI/ChatGPT account for authentication. Hermes uses Codex asone of the optional providersvia OAuth — can also run locally or with Anthropic. And it hasskill bundledcodexwhich delegates tasks to the Codex CLI as a sub-agent.

💰 Why Codex via Plus is strategic

  • ChatGPT Plus ($20/month):heavy use of non-pay-per-token coding. Those who program a lot are cheaper than API tokens.
  • OAuth on Hermes:reuse the subscription you already paid for.
  • Optimized models:GPT-5 codex, gpt-5 are specialized code.
  • Non-exclusive:you can switch between Codex (coding) and Anthropic (general reasoning) in the same Hermes session.

🎯 Crucial differences

CLI Codex:terminal-only, no cron, no skills, no gateway. Focus on code on the PC.
Hermes:it offers all of this on top and even uses Codex as an optional engine.
Strong combination:ChatGPT Plus + Hermes + skillcodex= coding with an optimized model, with gateway and cron on top.

🖱️ Hermes vs Cursor

Cursor isIDE with built-in copilot(inline autocompletion, side chat, agentic editing). Hermes is command line agent with gateway — runs onheadless server. Hermes detects and respects.cursorrules e .cursor/rules/*.mdc— the two are complementary, they do not compete.

🪽 Hermes wins in

  • ✓Non-graphical interface environments (VPS)
  • ✓24/7 automation via cron
  • ✓Cost (open source vs paid Cursor)
  • ✓Remote access via Telegram
  • ✓Multi-provider with Codex Plus

🖱️ Cursor wins in

  • ✓Interactive development in the IDE
  • ✓Real-time inline autocompletion
  • ✓UX integrated into the editor (Composer)
  • ✓Diff visual and change review
  • ✓Desktop-first workflow

🔄 Cursor + Hermes coexisting

This is the sweet spot for solo dev in 2026:Cursor produces code on the PC, Hermes operates infra on the server, both reading the same design rules.

🤝 How they coexist in practice

Shared rules: .cursorrules e .cursor/rules/*.mdcstay in the repo. Cursor reads locally. Hermes reads via plugin.
You on PC:open Cursor, do refactor, function autocomplete, local commit.
You outside the PC:send Telegram to Hermes — "deploy what's in main", "audit server", "reply to email X". Hermes operates everything headless.
Hermes Memory:records what was done; Cursor follows interactive context. No conflict.

💡Convention that works

Commit the rules to the repo. Both pull when logging in. Source of truth is Git. Rule changes propagate to both agents viagit pull.

🎛️ Codex CLI as Hermes sub-agent

Hermes comes with skill bundledcodexthat knows how to invoke the Codex CLI assubprocessfor heavy code tasks. You say "Hermes, use Codex to refactor this module" and it orchestrates: reads context, sets up the right prompt, calls Codex, receives diff, shows summary on Telegram.

🔧 The pattern "Hermes orchestrates, expert performs"

Skillcodex:delegates coding to Codex CLI with optimized OpenAI model.
Skillaider:delegates coding to Aider, with superior git integration (automatic commits, diff view).
Skillopencode:delegates to OpenCode, focus on software development.
Hermes becomes controller:decides which sub-agent to call for each type of task, maintains memory, communicates via Telegram.

📚 Skills bundled in Hermes (software-development category)

Hermes comes with several software development skills that know how to invoke specialized subagents. You get the best of each tool without having to decide in advance which one to use — the agent decides at runtime.

TOPIC 6
6

🎯 Decision: when to use each one

Direct answer:the four coexist with Hermes orchestrating. The choice of which to use changes by type of task, not ideological preference.

🌳 Practical decision tree

Interactive coding in the IDE with autocomplete?→ Cursor.
Coding via terminal sitting on the PC?→ OpenCode or Codex CLI.
Pair-programming with superior git integration?→ Aider.
Proactive agent via message with automations?→ Hermes.
All together in the routine?→ The four, with Hermes coordinating subagents.

💡Typical solo dev stack in 2026

Cursor + Codex CLI (subscription Plus) + Hermes (Telegram + cron). 100% charged on a single OpenAI subscription ($20/month) + a cheap VPS ($5-10/month). Total: ~$30/month covering IDE, CLI and agent.
