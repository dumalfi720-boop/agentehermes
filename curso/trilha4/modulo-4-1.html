🚫 Why NEVER paste tokens in chat

Everything you type in Hermes chat is persisted. The sessions are~/.hermes/state.db(SQLite with FTS5) and are searchable by the toolsession_search. If you paste a token there "just to test", itbecomes part of history forever.

⚠️What happens when you paste a token

  • •Go toIndexed SQLite— searchable for years
  • •Appears inautomatic backupsfrom VPS
  • •Can be sent toinference providerin compressions/summarizations
  • •Stay incontext of future sessionsvia session_search

🛡️What to do (and what NOT to do)

  • ✓Usehermes config set KEY valuealways
  • ✓If it was leaked by mistake:revoke credential immediatelyand generate another
  • ✗Don't try to "clear" your history — assume it's leaked
  • ✗Don't paste the token "just to test if it works"
TOPIC 2

⌨️ hermes config set KEY value

Official command to write environment variable without exposing it. It reads the value from stdin (or handled argv), persists it across/root/.hermes/.envinside the container and confirms without echoing the value.

📋 Common examples

hermes config set TELEGRAM_TOKEN 123456:abc...
hermes config set OPENROUTER_API_KEY sk-or-...
hermes config set OPENAI_API_KEY sk-...
hermes config set GITHUB_TOKEN ghp_...
hermes config set TELEGRAM_HOME_CHANNEL @meu_canal

🐳In Docker

Always execute inside the container before:

docker exec -it hermes hermes config set TELEGRAM_TOKEN 123:abc

Without theexec, you are running on the host — where Hermes is not even installed, or is in a different version.

🐳 container's .env vs host's .env

It's the community's #1 confusion. Docker isolates file systems. THE.envfrom the project root on the hostdoes not reach the processunless you mount it explicitly.

✗ What DOESN’T work

  • ✗Edit/home/voce/projeto/.envon host
  • ✗Wait for Hermes to "see" without volume mounted
  • ✗Restart the container and think it worked

✓ What works

  • ✓Volume~/.hermes:/root/.hermesmounted
  • ✓Variables in/root/.hermes/.envof the container
  • ✓hermes config setwrite in the right way

🔍How to confirm that you are right

docker exec -it hermes env | grep TELEGRAM

If the variable appears, it is visible to the process. If it doesn't appear, go back to the mount volume of thedocker-compose.ymlor in the var name.

📝 Edit/delete via nano

O config setoverwrites or adds, but has no subcommanddeletefriendly. When you change providers (left OpenAI, went to OpenRouter), the old key remains in the.env. Clean manually.

🔧 Safe procedure

  1. 1.
    Backup:docker exec hermes cp /root/.hermes/.env /root/.hermes/.env.bak
  2. 2.
    Edit:docker exec -it hermes nano /root/.hermes/.env
  3. 3.
    Delete the key line you no longer use
  4. 4.
    Restart gateway:hermes gateway restart
  5. 5.
    Confirm:docker exec hermes env | grep KEY_ANTIGA(should disappear)

💡Tip

Keep a comment at the top of.envwith date of last revision and who owns each key. In 6 months you won't remember whichAPI_KEYWhat is it for?

🎯 Tokens with limited scope

Leaked token is a loss proportional to the scope. Tokenclassicfrom GitHub with access to all your private repos = catastrophe. Tokenfine-grainedwith access only to repo X = contained incident.

🔑 GitHub: classic vs fine-grained

  • Classic:choose broad scopes (repo, workflow, gist...). No mandatory expiration. Access to all repos where you have permission.
  • Fine-grained:choose specific repos + granular permissions (read/write per area).Mandatory expiration(max 1 year).
  • Recommendation:always fine-grained, limited to minimum repos, with 90 days of expiration.

🤖 OpenAI/OpenRouter

  • OpenAI:useproject keys. Each project may have spending limits and its own models. Restrictsblast radius.
  • OpenRouter:create agent-named keys. Set monthly limit per key. Activate expense alert.
  • Anthropic:workspaces separate billing. Use one workspace per agent.

⚠️Rule of thumb

Start with scopeminimum. When the agent complains "permission denied", expandsówhat is necessary. Never the other way around (full access "because it's easier").

🔇 Self-writing secrets in logs

Hermes automatically redacts known values ​​of environment variables when they appear in error messages. You can ask for support, post log chunks on the community Discord — without leaking credentials.But always check before sharing.

📁 Where to look

  • ~/.hermes/logs/errors.log— agent errors, with self-writing
  • ~/.hermes/logs/gateway.log— messaging events
  • ~/.hermes/logs/cron.log— execution of scheduled jobs
  • ~/.hermes/state.db— sessions (usesession_search)

🛡️What the essay covers

  • ✓Variables with known names (TELEGRAM_TOKEN, *_API_KEY, *_SECRET)
  • ✓Recognized token patterns (sk-..., ghp_...)
  • ✗Hardcoded values ​​in code you passed in the chat
  • ✗Strings in free messages that look like regular text

💡Before sharing log

Always open and read. Search forsk-, ghp_, 123456:(Telegram token format), URLs with tokens in query string. Confirm that they are as***REDACTED***or similar.
