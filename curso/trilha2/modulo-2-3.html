🟢 Codex OAuth via ChatGPT Plus/Pro

The cheapest path for intense use in 2026.No API key.OAuth login via browser using your ChatGPT Plus ($20/month) or Pro ($200/month) account. Standard Model:gpt-5.3-codex, with vision included.

💰Why is it so cheap

  • ChatGPT Plus ($20):unlimited use within the plan's rate limit. Fits 10+ long conversations/quiet day.
  • ChatGPT Pro ($200):access to o3, o4-pro, GPT-5 with higher limits. For heavy professional use.
  • Direct API Comparison:Even using the API costs $100+/month. Plus saves 5x.
  • Without API key:authenticates via browser, token captured by Hermes — nothing to manage manually.

⚙️ Configuration

# config.yaml dentro do container (~/.hermes/config.yaml)
provider: "codex"
model: "gpt-5.3-codex"

# Login OAuth (abre URL no browser)
docker exec -it hermes hermes auth codex

⚠️Important limitation

"Unlimited" has an invisible rate limit. For very heavy use (hundreds of calls/hour), OpenAI can reduce speed or block temporarily. In normal personal use, you never touch the ceiling.

🌐 OpenRouter — flexible pay-per-token

OpenRouter is the most used aggregator: you put in credit once, choose any model (Claude, Gemini, DeepSeek, Grok, GPT) and pay per token consumed.Key advantage: exchange model per task, within the same provider.

🎁Featured models (May/2026)

  • openrouter/owl-alpha— free, good for light crons
  • google/gemini-2.5-flash— ~$0.075/1M tokens, ideal for context compression
  • deepseek/deepseek-v4-pro— added in v0.13.0, cheap top reasoning
  • x-ai/grok-4.3— added in v0.13.0, long answer
  • anthropic/claude-opus-4.6— premium quality, high cost
  • tencent/hy3-preview— added by contributor

⚙️ Configuration

# Setar API key
docker exec -it hermes hermes config set OPENROUTER_API_KEY sk-or-...

# config.yaml
provider: "openrouter"
model: "deepseek/deepseek-v4-pro"

# Compressão de contexto separada (opcional)
compression:
  provider: "openrouter"
  model: "google/gemini-2.5-flash"

💡Cost split strategy

Use top model (Claude/DeepSeek-Pro) for main conversation, and cheap model (Gemini Flash) for context compression. Hermes supports different provider per slot. Conversation is top-quality, compression is in cents.

🎭 Anthropic / Claude via API or GMI

When the task is critical — complex code review, long-term quality writing, strategic planning —,Claude Opus has superior qualityto any other open model or OpenAI in 2026. Cost is high, but worth it on time.

🛣️Two paths to Claude

  • Anthropic Direct API:account at anthropic.com,ANTHROPIC_API_KEY. Own billing.
  • Via OpenRouter: anthropic/claude-opus-4.6without needing to open an account with Anthropic. Small markup.
  • Via GMI:dealer,GMI_API_KEY. Useful for centralized billing in companies.

⚙️ Configuration via OpenRouter

provider: "openrouter"
model: "anthropic/claude-opus-4.6"

# Ou via Anthropic direto (requer ANTHROPIC_API_KEY)
provider: "anthropic"
model: "claude-opus-4.6"

✓When it's worth the cost

  • Refactoring or reviewing complex code (1k+ lines)
  • Professional long-form writing (articles, ebooks, technical documentation)
  • Strategic planning with analysis of multiple scenarios
  • Tasks where 5% extra quality changes the final result

🔌 MiniMax OAuth, Nous Portal, locations

Special cases:extreme privacy(sensitive data never leaves the network),zero cost for infinite volume(after hardware investment), oraccess to Chinese modelshigh quality via OAuth.

🌍 Less common but useful providers

  • MiniMax OAuth:browser login without API key, modelMiniMax-M2.7. Good quality for zero direct price.
  • Nous Portal: hermes auth + provider: "nous". NousResearch account (maintainer of Hermes).
  • OpenAI Direct API: OPENAI_API_KEY, useful if you already have OpenAI Enterprise credits.
  • Location (Ollama, LM Studio, vLLM):any OpenAI-compatible endpoint. Setup:base_url: "http://host.docker.internal:11434/v1".

💻 Local endpoint example

# Mac Mini com Ollama rodando Qwen ou Llama
provider: "openai"
base_url: "http://192.168.1.10:11434/v1"
model: "qwen2.5:14b"
api_key: "ollama"  # placeholder, Ollama não valida

🧠Tool-use enforcement

Models withself enforcement(Hermes injects tool-use reinforcement):gpt, codex, gemini, gemma, grok. Natively trusted models (without extra reinforcement):Claude, DeepSeek, Qwen, LLaMA. Knowing this helps when a tool is not being called.

🎯 How to decide: criteria table

Without criteria, you keep changing providers all the time. With clear criteria, you choose once and adjust over time. Switching is trivial in Hermes (one line inconfig.yaml), but jumping for no reason burns mental energy.

🌳 Decision tree

  • Already have ChatGPT Plus/Pro?→ Codex OAuth is the cheapest way. Start there.
  • Want flexibility between models without multiple accounts?→ OpenRouter as hub.
  • Premium tasks (writing, critical analysis)?→ Claude Opus punctual via OpenRouter.
  • Extreme privacy (sensitive data)?→ Local via Ollama (Mac Mini or desktop with GPU).
  • Does giant volume and cost matter?→ DeepSeek/Gemini Flash via OpenRouter.
  • No credit card?→ MiniMax OAuth or free OpenRouter models.

🔄How easy is it to switch

Migration between providers takes 30 seconds: edit~/.hermes/config.yaml + docker compose restart. Skills, memory, sessions — everything remains. Use Codex for 3 months, switch to OpenRouter, switch to local — zero pain. This means:Don't procrastinate on your first choice. Start with Codex OAuth, optimize later.

💡Common mistake

Try to configure 4 providers simultaneously "to have options". Result: none working properly, parallel debugging becoming a snowball. Start with ONE. Dominate. Add the second only when the first is solid.
