🚀 Why start with Telegram

Hermes supports20+ messaging platforms— Discord, Slack, WhatsApp, Signal, Teams, Matrix, iMessage. But Telegram is where you should start.Setup in 5 minutes versus 30+ minutes for others.

⚡Friction comparison

  • Telegram:@BotFather in chat, 30 seconds for the token. No extra registration.
  • Discord:developer portal, create app, create bot, configure intents, generate invite link.
  • Slack:create app at api.slack.com, configure OAuth scopes, install in workspace.
  • WhatsApp:via paid gateway (Whapi) or Meta Business Manager — bureaucratic.
  • iMessage:requires Mac at home running BlueBubbles 24/7.

📦What do you get for free?

  • Voice notes that Hermes transcribes and responds to with TTS
  • slash commands/cron, /usage, /skills, /compress
  • Image sharing (automatic vision)
  • Sharing PDF, audio, video files
  • Synchronized desktop and mobile client

🤖 Create bot with BotFather

BotFather is an official bot from Telegram itself that creates other bots. No developer portal, no approval, no registration.5 messages exchanged in chat and you have the token.

📝 Complete sequence

  1. On Telegram, search@BotFatherand open the chat.
  2. Send/newbot.
  3. He asks whatname (display): eg. "Personal Hermes".
  4. He asks whatusername: must end inbotand be unique globally. Ex.:hermes_pessoal_xyz_bot.
  5. He returns thetoken: something like1234567890:AAH.... It is the bot's full credential.

🔒Token handling

  • Token = bot password. Whoever has it controls it completely.
  • NeverPaste in public chat, forum, repository, GitHub, screenshot.
  • Usehermes config set TELEGRAM_TOKEN xxxinside the container — writes with auto-redaction to the logs.
  • If it leaks:/revokein BotFather generates new token and invalidates the old one.

🎨Useful customizations in BotFather

  • /setdescription— text that appears when someone opens the bot
  • /setuserpic— profile photo
  • /setcommands— list of slash commands that appear on Telegram
  • /setprivacy— in groups, defines whether the bot sees all messages or only @mentions

🆔 Get your user ID via @userinfobot

Hermes needs to know who is authorized to speak to him. Without this configuration, the bot either accepts anyone (dangerous) or ignores you (frustrating). The stable form of identification is thenumeric user ID— not the username, which can change.

🔍 How to find out your ID

  1. On Telegram, search@userinfobot.
  2. Send any message (or/start).
  3. He responds with hisNumeric ID(ex:123456789), name, language, etc.
  4. Make note of this ID — it is used inTELEGRAM_AUTHORIZED_USERS.

👥Group and channel IDs

For groups, the ID starts with-100...(negative). Add the bot as a member and use@RawDataBotor similar to capture the group ID. Useful for using Hermes in a shared chat with a spouse/partner.

💡Multiple authorized users

TELEGRAM_AUTHORIZED_USERSaccepts comma separated list:123456789,987654321. Useful for sharing agents with a small team — but remember that everyone has full access. For larger teams, separating by agent is safer.

TOPIC 4

⚙️ hermes gateway setup

With token and user ID in hand, it's time to connect everything.hermes gateway setupopens an interactive wizard inside the container — guided, with validation at each step.

🪄 Step-by-step wizard

# Entre no container
docker exec -it hermes hermes gateway setup

# O wizard pergunta:
# 1) Qual plataforma? → telegram
# 2) Cole o bot token: → 1234567890:AAH...
# 3) Cole user ID(s) autorizado(s): → 123456789
# 4) Home channel (canal padrão pra crons): → 123456789

# Depois, suba o gateway
hermes gateway start

📂What does the wizard save where

  • ~/.hermes/.env— sensitive variables (TELEGRAM_TOKEN)
  • ~/.hermes/config.yaml— gateway settings, authorization
  • TELEGRAM_HOME_CHANNEL:default destination for automatic deliveries (crons without--deliver)
  • TELEGRAM_AUTHORIZED_USERS:list of IDs that can send commands

🔄 Related commands

  • hermes gateway status— is gateway running?
  • hermes gateway start— raise the gateway (or re-up after restart)
  • hermes gateway stop— to the gateway
  • hermes gateway setup— re-run wizard if you want to adjust
TOPIC 5

💬 First conversation + debug

Time of truth. Open the chat with your bot on Telegram, send"Hello". Expected response: the agent responds by introducing himself.If it passes, the entire setup is OK.If it fails, the logs show the objective error.

🐛 Debug commands

# Status do gateway
docker exec hermes hermes gateway status

# Logs em tempo real
docker exec hermes tail -f /root/.hermes/logs/gateway.log

# Verificar variáveis carregadas
docker exec hermes env | grep TELEGRAM

# Testar conexão com API do Telegram
docker exec hermes curl -s https://api.telegram.org/bot$TELEGRAM_TOKEN/getMe

🚨Most common errors

  • "401 Unauthorized":wrong token, with extra space or extra copied character. Redo config set.
  • "user not authorized":your ID is not inTELEGRAM_AUTHORIZED_USERS. Check it out@userinfobotand redo setup.
  • "connection timeout":VPS firewall blocking egress toapi.telegram.org. Checkufwoutgoing.
  • No response or error:gateway may be stopped.hermes gateway start.
  • Bot responds but takes a while:credit-free or rate-limited inference provider. Check/usage.

✓Applying the T1.2 mindset

If nothing worked, copy the error from the log and paste it into the Hermes CLI (docker exec -it hermes hermes): "This error appeared when I tried the gateway. Diagnose and propose correction."He reads the doc, identifies the symptom, suggests fixing it.

🎚️ Customization: voice, slash, permissions

Connection working. Now it’s worth understanding 3 capabilities that greatly change the experience:voice notes, slash commandsand the system ofpermissions(allow once/session/always).

🎤 Voice notes

Press and hold the microphone on Telegram, speak, release. Telegram sends audio. Hermes transcribes, processes, and can respond with text OR with TTS back. Useful on the move — walking, driving, cooking.

⌨️ Most useful slash commands

  • /cron list— all scheduled jobs
  • /cron add "every 1d at 09:00" "Resumo diário"— create cron
  • /skills— list of available skills
  • /usage— how many tokens spent in the session
  • /compress— force context compression
  • /help— quick guide within the chat

🔐Permissions system

  • Allow once:just this execution. Good for single destructive action (delete file, force push).
  • Allow session:within the current session. Good for exploration ("I'm going to make several edits now").
  • Allow always:persistent in~/.hermes/permissions.json. Good for safe recurring actions (git commit/push in your repo).

⚠️Use "always" judiciously

Marking everything as "always allow" overrides the protection system. Use only for repeated actions, in a locked scope, with low risk. Audit regularly:hermes permissions list.
