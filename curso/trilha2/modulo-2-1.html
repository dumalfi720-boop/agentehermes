🤔 Why VPS and not personal laptop

Hermes can run on your laptop.But you won't wantonce you understand the trade-off. A personal machine was made for you to use — close the lid, sleep, travel, reboot. An agent that serves crons and responds to Telegram needs the opposite:always be connected, with stable IP, permanent internet and zero supervision.

⚖️VPS vs Laptop — direct comparison

  • Uptime:VPS runs 24/7 with SLA. Laptop sleeps when you close it.
  • IP:VPS has fixed public IP. Laptop changes every time it changes Wi-Fi.
  • Cost:VPS ~$9/month. Laptop running 24/7 consumes electricity and fans.
  • Maintenance:VPS isolated from your personal use. Crash on the laptop takes the agent down with it.
  • Access:VPS via SSH from anywhere. Laptop needs to be physically accessible.

💰Cost of not having a VPS

Imagine backup cron failing 3 days in a row because the laptop was turned off. Monitoring cron missing 2 critical alerts. Telegram response arriving 6 hours later.The cost of "saving" $9/month is chronic pain.

🐧 Operating system: Ubuntu 24.04 LTS

When deploying, chooseUbuntu 24.04 LTS. It's the community standard — all tutorials use it, guaranteed support until 2029, active community, updated repositories. Alternative distros (Debian, Alpine, CentOS) have gotchas with Docker, NodeJS, Python that cost time.

🔧Initial settings in the dashboard

  • Hostname:choose something memorable (hermes-pessoal, hermes-prod). Appears at the shell prompt.
  • Datacenter:choose the closest one (São Paulo if Hostinger offers it, otherwise Miami).
  • Automatic backup:ACTIVATE. It costs ~$1/month more and saves your life when something goes wrong at 11pm on a Friday.
  • Root password:generate strong (passphrase 4+ words) — you will only use it once, then replace it with SSH key.

⚠️What NOT to choose

  • ✗CentOS/Rocky/AlmaLinux:fewer tutorials, different package manager (yum/dnf), incompatibility with community scripts.
  • ✗Alpine:too minimalistic — lacks complete libc, problems with precompiled Python binaries.
  • ✗Non-LTS Ubuntu:23.10, 24.10 lose support in 9 months. LTS lasts 5 years.
  • ✗"With Docker pre-installed" images:outdated versions. Prefer to install via official script.

🔑 First SSH and basic commands

Server provisioned, IP in hand. Time for the first SSH and the update + user creation routine. These 4 commands are almost universal for any fresh Linux VPS.

📝 Starting Sequence

# 1. Conectar
ssh root@SEU_IP

# 2. Atualizar tudo
apt update && apt upgrade -y

# 3. Criar usuário não-root e dar sudo
adduser hermes
usermod -aG sudo hermes

# 4. Firewall mínimo (UFW já vem no Ubuntu)
ufw default deny incoming
ufw default allow outgoing
ufw allow 22/tcp        # SSH
ufw enable
ufw status

✓Why Every Command Matters

  • apt upgrade:applies outstanding security patches to the default image (there are always some).
  • Non-root user:Running Hermes as root expands blast radius. Agent error in/etcdestroys the server.
  • UFW deny incoming:blocks everything by default. You only open what is necessary.
  • Allow 22 before enable:forgetting this order cuts off your own access. This error brings down about 20% of newbies.

🛡️ Basic hardening before Hermes

Public server in 2026 takes thousands of login attempts per day. Without hardening, it's just a matter of time before someone joins. With 3 simple measures —SSH key required, root login disabled, fail2ban— the server is practically untouchable by brute-force.

🔐 Hardening in 3 steps

# 1. Gerar chave no SEU LAPTOP (não no VPS!)
ssh-keygen -t ed25519 -C "hermes-vps"

# 2. Copiar pro VPS
ssh-copy-id hermes@SEU_IP

# 3. Editar /etc/ssh/sshd_config no VPS
sudo nano /etc/ssh/sshd_config
# Mude:
#   PasswordAuthentication no
#   PermitRootLogin no
sudo systemctl restart sshd

# 4. Instalar fail2ban
sudo apt install -y fail2ban
sudo systemctl enable --now fail2ban

⚠️Before closing the session

NEVERrestart sshd and close the session without testing. Open another terminal window, try connecting with the key. If it works, OK. If it breaks, you still have the original session open to undo. This habit saves many hours (and the cost of requesting a reset on the Hostinger panel).

💡Advanced (optional, for later)

  • Change SSH port:from 22 to something random (ex: 2222) reduces 99% of automatic scans.
  • Tailscale:hides SSH behind mesh VPN, public IP can be closed to the world.
  • Cloudflare Tunnel:alternative to exposing HTTP services without opening a port on the firewall.
