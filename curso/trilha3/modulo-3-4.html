🚀 Reactive → proactive: fresh session per run

Cron is what turns Hermes into an assistantproactive. Without cron, it only responds when you speak. With cron, it triggers at defined times, opensfresh sessionfrom AIAgent, executes the prompt and delivers the result.

⚙️How each tick works

  1. Load jobs from~/.hermes/cron/jobs.json
  2. Checknext_run_atvs current time
  3. Start a fresh AIAgent session for each job due
  4. Injects attached skills (--skill)
  5. Run the prompt until completion
  6. Deliver the final response to the destination (--deliver)
  7. Update metadata and next scheduled time

⚠️Session fresh = self-contained prompt

The session is new with each shot. There is no previous conversation history. SOUL.md, USER.md, and MEMORY.md are loaded, but any context beyond that needs to be at the cron prompt."Continue what you were doing"It doesn’t work — there’s nothing to go on.

🔒File lock prevents double-run

The scheduler writes to~/.hermes/cron/.tick.lockevery tick. If a tick takes longer than the interval, the next one does NOT fire in parallel — wait. This avoids running the same task twice at the same time (classic in integration).

💬 Natural syntax via chat

The fastest way to create cron is directly via chat (Telegram, Discord, terminal). Command/cron addaccepts expressions innatural language.

📝 Ready-made examples

/cron add 30m "Remind me to check the build"
/cron add "every 2h" "Check server status"
/cron add "every 1h" "Summarize new feed items" --skill blogwatcher
/cron add "every 1h" "Use both skills" --skill blogwatcher --skill maps
/cron add "every 1d at 09:00" "Resumo do dia + agenda"

Relative durations

  • 30m, 2h, 1d→ in X amount of time
  • Useful for occasional reminders

Natural recurrence

  • "every 2h"→ recurrence
  • "every 1d at 09:00"→ diary with schedule
  • Useful for routines

💡Useful commands

  • • /cron list— see active crons
  • • /cron remove <id>— remove cron
  • • /cron pause <id> / /cron resume <id>

🖥️ Standalone CLI: hermes cron create

Same capacity via terminal. Useful forprovisioning scripts(infra-as-code) that recreate all crons on a new machine at once.

📝 CLI Example

hermes cron create "every 1d at 09:00" \
  "Audit open PRs, summarize CI health, and post to #eng" \
  --workdir /home/me/projects/acme

📦setup-crons.sh versioned

Create a scriptsetup-crons.shin your repo below. Each cron becomes a line. Migrate to new VPS = run the script. Reproducible, auditable, clickless.

#!/usr/bin/env bash
hermes cron create "every 1d at 09:00" "Resumo diário" --deliver telegram
hermes cron create "every 6h" "Backup S3" --workdir ~/projetos/site
hermes cron create "every 5m" --no-agent --script ./watchdog.sh \
  --deliver telegram --name "memory-watchdog"

📁Where it is

Settings persist in~/.hermes/cron/jobs.json. You can version this file too — natural backup without having to redo setup. Subcommands:hermes cron list, hermes cron remove <id>.

📂 Flag --workdir <path>

Sets theworking directoryfrom the fresh session. The scheduler doescdthere before running and detects the project context files.Traditional cron equivalent: WORKDIR.

📚Project context priority order

  1. .hermes.md ou HERMES.md(goes up to git root)
  2. AGENTS.md(recursive walk of subdirectories)
  3. CLAUDE.md(CWD only)
  4. .cursorrules(CWD only)

JustUMtype is loaded per session. SOUL.md always loads independently.

⚠️Without --workdir, it runs in ~

Cron for a specific project needs the context of that project. Without--workdir, the scheduler runs in~(home) and does not find .hermes.md/AGENTS.md/CLAUDE.md of the project. Result: lost agent, without code conventions or specific instructions.

📝Practical example

hermes cron create "every 1d at 18:00" \
  "Audit changed files today, propose changelog entry" \
  --workdir /home/me/projetos/api-clientes \
  --deliver telegram

The session reads the project's AGENTS.md and follows specific conventions (changelog format, branch policy, etc.).

TOPIC 5
5

🔇 Flag --no-agent (NOAGENT mode)

So thatskip the LLM completely. Cron only runs one script.sh/.bash/.py. Empty output = silence. Non-zero output = alert delivered. Ideal for watchdogs who don't need reasoning.

📝 Memory Watchdog

hermes cron create "every 5m" \
  --no-agent \
  --script memory-watchdog.sh \
  --deliver telegram \
  --name "memory-watchdog"

empty stdout

Silent tick, no delivery. Ambience is OK.

non-empty stdout

Content is delivered as an alert to the configured destination.

non-zero exit

Same thing: error alert goes to the destination. Good for detecting failure of the script itself.

⚙️How the script is executed

  • • Scripts.sh / .bashrotate under/bin/bash
  • • Others (Python etc.) run on Hermes' Python interpreter
  • • Works with any language that produces standard stdout/exit code

💰Why NOAGENT

Memory/disk watchdog runs every 5min. If I went through the LLM, I would spend tokens unnecessarily. NOAGENT maintains zero costs for this type of task while preserving delivery on the messaging platform.

TOPIC 6
6

📡 Flags --deliver and --skill

Two flags that complete the picture:--deliver <plataforma>choose destination.--skill <name>injects skill at the beginning of the fresh session, ensuring that the procedure is available even without history.

--deliver <platform>

  • telegram— send it to the main Telegram chat
  • discord— configured Discord channel
  • all— all active platforms
  • origin— where cron was created

--skill <name>

  • Load skill at the beginning of the fresh session
  • You can pass multiple:--skill A --skill B
  • Ensures available procedure without relying on automatic discovery

📝 Example combining everything

hermes cron create "every 1h" \
  "Summarize new feed items and post the digest" \
  --workdir /home/me/projetos/blog \
  --skill blogwatcher \
  --skill maps \
  --deliver telegram

🎯Flag summary

FlagCron equivalentFunction
--workdir <path>WORKDIRWorking directory + project context
--no-agentNOAGENTScript-only, skip LLM
--deliver <plat>CONTEXTFROMDelivery platform
--skill <name>skill injectionInject skill into the fresh session
