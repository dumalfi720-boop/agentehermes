📜 What is a skill

Skill is a fileSKILL.mdwithYAML front matterat the top and structured instructions below. It's areusable procedurewritten in natural language — the model reads and follows.

🆚Skill vs prompt vs code

  • Single prompt:disappears at the end of the session. Always repeat.
  • Skill:Structured Markdown, persisted, discovered by the agent when relevant.
  • Code:deterministic but inflexible. Does not make decisions based on context.

📡Open standard

Hermes adopts the public specagentskills.io/specification. Skill created for Hermes is also valid for other agents that adopt the standard. Hub Discord: discord.gg/MKPE9g8aUy.

📋 Anatomy of YAML front matter

The front matter is the YAML header between---that describes the skill. It is read at each session to create the list that the agent consults. Bad description = skill never called.

📝 Front matter complete

---
name: my-skill
description: Brief description of what this skill does
version: 1.0.0
platforms: [macos, linux]     # Opcional — restringe a OS específicos
metadata:
  hermes:
    tags: [python, automation]
    category: devops
    fallback_for_toolsets: [web]    # Opcional — ativação condicional
    requires_toolsets: [terminal]   # Opcional — ativação condicional
    config:                         # Opcional — config.yaml
      - key: my.setting
        description: "What this controls"
        default: "value"
        prompt: "Prompt for setup"
---

Required fields

  • name— unique identifier
  • description— what does it do, in one line
  • version— semver

metadata.hermes (optional)

  • tags— quick search
  • category— grouping
  • fallback_for_toolsets— activates if another fails
  • requires_toolsets— requires available toolset
  • config— setup prompts

🧱Standard body structure

Below the front matter comes the Markdown content. The model waitsfour canonical sections: When to Use, Procedure, Pitfalls, Verification. Skipping any one degrades the quality of the skill.

📝 Canonical skeleton

# Skill Title

## When to Use
Trigger conditions for this skill.

## Procedure
1. Step one
2. Step two

## Pitfalls
- Known failure modes and fixes

## Verification
How to confirm it worked.

When to Use

Triggers. When the agent should invoke this skill. Without this, he either never calls or calls wrong.

Procedure

Numbered, direct steps. One step per line. Ideally verifiable.

Pitfalls

What has already broken and how to get around it. Collected from real experience, not imagination.

Verification

Short test: "if you run X, Y should come out". Without this, the agent declares success, thinking it worked.

🪜 Progressive Disclosure (3 levels)

The great efficiency trick: the agentdoes not load all skills in system prompt. Load only a minimum list and open each skill on demand. Pattern calledProgressive Disclosure.

Level 0 —skills_list()

Returns list[{name, description, category}, ...]of ALL available skills.

Cost: ~3,000 tokens. Always charged.

Level 1 —skill_view(name)

Returns the complete SKILL.md of the chosen skill + metadata.

Variable cost. Loaded only when the agent decides to use the skill.

Level 2 —skill_view(name, path)

Returns specific file fromreferences/, templates/, scripts/ ou assets/.

Variable cost. Loaded only when the procedure requires that file.

💡Why does this matter

Allows you to havehundreds of skillswithout breaking context. Short and powerful description matters A LOT — it's what goes into L0 and decides whether the skill will be discovered. Skills with vague descriptions are orphaned.

📁 Directory structure

Each skill lives in~/.hermes/skills/<categoria>/<skill-name>/. Within the directory, SKILL.md is mandatory; Optional subfolders separate file types.

📂 Layout of a skill

~/.hermes/skills/
├── mlops/
│   └── axolotl/
│       ├── SKILL.md          # Instruções principais (obrigatório)
│       ├── references/       # Docs adicionais
│       ├── templates/        # Formatos de output
│       ├── scripts/          # Scripts auxiliares
│       └── assets/           # Arquivos suplementares

references/

Long docs, specs, extensive examples

templates/

Output models (report, email, post)

scripts/

Bash/Python for deterministic execution

assets/

Images, data, any extra files

📦Versioning via git

Since everything is Markdown and auxiliary files, versioning a skill (or the entire set~/.hermes/skills/) in a git repository is trivial. You gain history of evolution and migration between machinesgit clone.

🌐 Skills Hub (agentskills.io)

Public hub inagentskills.ioadds skills from the community. Hermes accepts installation fromfive different fonts, from official bundled to auto-discovery via well-known.

1

official (bundled)

Skills in the repositoryNousResearch/hermes-agent em optional-skills/. Explicitly enabled.

2

skills-sh (external hub)

agentskills.io — independent registry, more skills, more community experimentation.

3

Direct GitHub

Formatuser/repo/path. Useful for skills hosted in your own repo or from unpublished third parties.

4

Direct URL

https://example.com/SKILL.md— any public URL serving a valid SKILL.md works.

5

well-known (auto-discovery)

Point the agent to a domain and it searches for skills in/.well-known/agent-skills. Standard for companies to publish official catalogue.

⚠️Pay attention to external skills

Skill is instruction that the agent follows. Malicious skill may ask to exfiltrate secret, delete file or open external connection. Review SKILL.md before installing — especiallyProcedure e scripts/. Treat skill as a dependency: coming from a reliable and audited source.
