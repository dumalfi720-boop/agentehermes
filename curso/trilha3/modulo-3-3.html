👻 Soul: agent identity

TOPIC 1

🎭 SOUL.md in slot #1 of the system prompt

SOUL.md is thefirst blockfrom system prompt — before memory, before skills, before everything. It is the basis of the identity that the model reads for any interaction. Always loaded, regardless of context.

📐System prompt order

  1. SOUL.md— agent identity (always)
  2. Memories (USER.md + MEMORY.md, snapshot frozen)
  3. Project context (.hermes.md / AGENTS.md / CLAUDE.md / .cursorrules — only ONE, in priority order)
  4. Minimum list of skills (Level 0 of Progressive Disclosure)
  5. Operating instructions and available tools

🎯Why position matters

Templates tend to give more weight to what appears early in the system prompt. Identity in slot #1 shapes how he interprets everything that comes after. Wrong tone there echoes in every answer.

TOPIC 2
2

📍 Location: ~/.hermes/SOUL.md

Single file in~/.hermes/SOUL.md. Scopeglobal: applies to any project, any directory.Pure Markdown, without YAML front matter (unlike SKILL.md).

Features

  • • Fixed location:~/.hermes/SOUL.md
  • • Global scope (does not change with cwd)
  • • Pure Markdown
  • • No YAML front matter
  • • No documented character limit

Good practices

  • • Keep it concise (~500-1000 words)
  • • Version in git to have history
  • • Review before accepting agent rewrite
  • • Back up your~/.hermes/whole

🎭Multiple profiles = multiple souls

To have agents with different identities (cheerful marketing, serious finance), usedifferent Hermes profiles: each profile points to its own directory.

~/.hermes-marketing/SOUL.md   # alegre, emojis, tom de voz da marca
~/.hermes-financeiro/SOUL.md  # sóbrio, direto, sem floreios
~/.hermes-pessoal/SOUL.md     # informal, próximo, contextual
TOPIC 3
3

🎨 What SOUL defines

Bom SOUL.md covers four axes:tone, personality, principles e communication rules. The more specific, the more characteristic the agent.

🎚️ Tom

How he expresses himself. Formal or relaxed? Direct or explanatory? Technical or affordable?

Example:"I speak directly, I avoid formalism. I don't use superlatives without evidence. I prefer short sentences."

🧬 Personality

Who the agent is. Curious? Skeptical? Pragmatic? Provocative? Mentor?

Example:"I'm a pragmatist. I prefer a simple solution that works over an elegant solution that hasn't been tested."

⚖️ Principles

What you always do, what you never do. Hard lines that are not negotiated by context.

Example:"I always cite a source of technical fact. I never invent a URL. When I don't know, I say I don't know."

📞 Communication rules

Language, default response size, use of emojis, list format, fence code.

Example:"PT-BR by default. Short answer first, detail if asked. No emojis in technical answers."

⚠️Beware of contradictions

"Be succinct" + "Explain each decision in detail" fight. "Use emojis to keep things light" + "Maintain a professional tone without flourishes" fight. Soul needs to be coherent as a character — internal contradiction degrades answers.

TOPIC 4
4

🎯 Why multi-agent matters

In setup with multiple profiles (personal, marketing, financial, ops),soul is what gives a distinct voiceto each one. Without their own soul, everyone sounds the same and loses the sense of existing separately.

Marketing

Soul: light, curious, likes to explore trends, brand tone. You can use emojis. Knows copywriting.

Financial

Soul: sober, direct, conservative, quotes numbers. No emojis. Accuracy matters more than agility.

Guys

Soul: informal, gets to know you, contextual. You can play. Focus on usefulness and your time.

🎨The complete trio

  • Souldefinevoice(as he speaks)
  • Skillsdefinecapabilities(what you know how to do)
  • memoryloadfacts(what do you know about the environment)

The three combined give complete character to the profile. Lack of either leaves the agent "shallow".

TOPIC 5
5

🔄 How to evolve SOUL

You don't need to become a prompt designer. Magic phrase:"Upgrade your soul to be X". The agent himself edits SOUL.md — and it works surprisingly well because he understands the effect of each line on himself.

💬 Ready-made phrases

  • • "Read your current soul and show me."
  • • "Update your soul: be more direct, less explanatory."
  • • "In your soul, add rule: never use exclamation in technical response."
  • • "Rewrite your soul with our brand tone: <example text>."
  • • "Review your soul, identify contradictions and propose correction."

⚠️Always review before accepting

Soul is a critical file — a bad rewrite degrades the agentall. Ask to show the diff before saving. Keep versioned in git to easily revert. If it looks strange after editing, scroll back to the previous commit.

📅Review cadence

Review soul every 1-2 months. Things that made sense at the beginning can become obstacles later. And new things emerge in real use — ask the agent to incorporate lessons learned from the last few weeks.

TOPIC 6
6

⚖️ Soul vs USER.md

Confusing SOUL.md with USER.md is the most common mistake. The rule is simple:SOUL = who the agent is. USER = who you are. Mixing causes strange things (agent thinking HE is Nei, or personal rules disappearing in the wrong slot).

SOUL.md (the agent)

  • ✓"I'm direct and practical."
  • ✓"I don't use emojis in technical responses."
  • ✓"When I make a mistake, I admit it bluntly."
  • ✓"I'm skeptical of claims without evidence."

USER.md (you)

  • ✓"Nei, programmer, speak PT-BR."
  • ✓"Prefer short answers with example."
  • ✓"Works with Python and n8n."
  • ✓"Fuso: America/Sao_Paulo."

🧪Mental test

"If I changed the user, would this still be valid?"

  • • If so→ go toSOUL.md(it's the agent's trait)
  • • If not→ go toUSER.md(it's given yours)
