MODULE 6.4

🔁 Continuous maintenance routine

The difference between 1 month and 1 year of Hermes is routine. Simple rules, periodic reading, debugging using Markdown before code, rotation cadence and the philosophy that keeps the agent alive.

📐 Simple rules

Maintenance without rules becomes "when time permits" — and it never does. With a simple rule, it becomes a reflex. Here are thethree that cover 90% of cases:

1. Missed 2× → update skill or memory

First time is chance, second time is standard. When the agent makes the same mistake twice, you open the skill (if it is a procedure) or the memory (if it is a fact) and correct it immediately.

2. Repeated instruction → becomes skill

If you're giving the same guidance ("do X, then Y, then Z") for the 3rd time, it's time to save it as a skill."This has become a skill, save."The agent takes care of the rest.

3. Wrong tone → adjusts soul

When the agent "goes out of character" — too formal, too informal, cold when he should be warm — it's not a bug, it's soul.md outdated or ambiguous. Adjust 1-2 lines and test.

💡Magic phrase

"This has become a skill, save."The agent creates SKILL.md in the correct format, saves to~/.hermes/skills/, and on the next invocation it is now available. You don't need to remember the syntax.

📖 Periodic reading

Monthly, ask the agentaudit yourself. It's a cheap audit: you see in 5 minutes what is outdated, duplicated or orphaned. Without this, memory becomes a silent dump.

📋 Monthly orders

  • •"Read your memory and tell me what seems out of date"
  • •"Read your soul and tell me if it still reflects how I should behave"
  • •"List your active skills and indicate 3 that have not been used in the last month"
  • •"List active crons and say which ones still make sense"

⏰Automate with cron

hermes cron create "1st of every month at 9:00" \
  "Faça auditoria de memória, soul, skills e crons. \
  Liste 3 itens pra podar. Aguardo aprovação." \
  --deliver telegram

You just approve or reject it. Maintenance without your deliberate effort, but with you in the loop for decisions.

TOPIC 3
3

🧟 Stale memory is #1

Outdated memoryis the main cause of Hermes' "strange behavior". Old VPS that migrated, dead project that continues to be mentioned, preference that changed. The agent follows the memory — and it seems "wrong."

⚠️Stale memory symptoms

  • •Agent mentions tool you no longer use
  • •Suggests meetings/contacts with old people
  • •Wrong preferences ("you like X" when you now prefer Y)
  • •Wrong file paths (old server, moved folder)
  • •Inappropriate tone for the current phase of your life/project

📓Memory as diary, not log

Memory.md should be edited asdaily, with a current version of the facts — not aslogthat accumulates everything. When something changes in your life, ask:

"Updates memory: I now use X instead of Y. Removes old references to Y."

🔍 Before any debugging

70% of Hermes' strange behaviors areOutdated fact in memory.md. Before assuming code bug, open the file. It's the cat's leap.

TOPIC 4
4

🔍 When something breaks: memory.md first

Hermes is90% Markdown. Skills, memory, soul, configs. Errors are often old or ambiguous instructions, not bugs. Have a clear debugging protocol that starts with the text.

📋 Debug protocol

  1. memory.md— outdated fact is the most common cause
  2. soul.md—ambiguous tone generates erratic response
  3. skill involved— outdated or conflicting procedure with another skill
  4. logs— just now open~/.hermes/logs/
  5. config— config.yaml, .env, plugins
  6. code— last frontier; is rarely the problem

💬Phrase to delegate debugging

"Strange behavior inX. Audits memory, soul and related skill before I change the code. Let me know what you found inconsistent."

The agent does the audit work. You review and approve changes. Saves hours spent debugging code.

💡 "Markdown first" philosophy

Hermes' design intentionally puts knowledge into editable text, not hidden bank. This means thattext is the first place to look. When you incorporate this, debugging becomes cheaper and faster.

TOPIC 5
5

🔐 Rotation and audit

Without cadence, no one performs safety maintenance. With cron, it becomes automatic: the agent remembers you. It's the difference between"I intend to" e "it's happening".

🔑 Quarterly: token rotation

Telegram, Gmail, GitHub, OpenRouter, Anthropic. Every 3 months, generate new ones and revoke old ones. Cron reminds you every 90 days with a checklist of what to rotate.

🧹 Monthly: skills review

Monthly Cron asks the agent: "list skills that have not been used for 30+ days, with suggestions for pruning". You approve archiving or deletion.

🛡️ Weekly: security audit

Skill bundled "security audit" checks open ports, new processes, disk space, abnormal logs, SSH keys. Cron every Monday morning.

🎯 Why Expired Tokens Matter

Revoked Telegram token is a frequent cause of "mysteriously stopped gateway". Expired inference provider token may stop all crons. Programmed rotation avoids chaos.

TOPIC 6

🤝 Philosophy: an agent is a colleague who learns

Here's the philosophical summary that carries it all. HermesIt is not a tool that ends— it's not like an app that you install and it's ready. He's a colleague: he needs onboarding, context, correction and growth.

🎭The two ways of treating

❌ As a static tool

"I installed it and it's ready." It doesn't correct, it doesn't update, it doesn't train. In 60 days, it becomes "that agent that I have but I don't use much".

✅ As a colleague who learns

Careful onboarding, corrections when mistakes are made, updates when context changes. In 60 days, the agent is more useful than on day 1.

👤Ultimate mindset

You are thetech lead. He is thesenior intern. Each has a clear role:

  • •You:approves, corrects, teaches, sets priorities, audits
  • •He:executes, remembers, suggests, automates, asks what you don't know

Every month, more alignment. Every correction, more intelligence. Each new skill, more autonomy.

🌱What do you take

If there is only one thing to carry out of these 6 trails:Agent Hermes is not a project that ends. It's a relationship that grows.Anyone who understands this has an increasingly useful agent. Those who don't understand will leave within 60 days.
