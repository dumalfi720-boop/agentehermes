🖥️ What's on the dashboard

The Hermes web UI (moduleweb/in the repository) is a complete dashboard. It's not a gimmick — it's a serious administration interface with everything you need to operate multiple agents.

💬 Sessions

Complete conversation history, with FTS5 search. Reopen old session, export transcript, prune stale conversations.

📡 Channels

List of active messaging platforms (Telegram, Discord, etc), gateway status, messaging metrics per channel.

⏰ Crons

All active schedules, next run, run history, buttons to pause/edit/duplicate cron.

🛠️ Skills

Skills bundled, installed via hub, and local custom. Inline SKILL.md editor. Usage indicator (how many times invoked).

🔌Plugins

LCM, Kanban, community integrations. Version, status, update/disable with 1 click.

🗒️ Kanban

New in v0.13.0. Boards with task cards, swim lanes per worker, heartbeat and zombie indicators.

📌 Important

Everything the dashboard shows is also in text files in~/.hermes/. UI is visual convenience, not hidden banking. You can edit everything via the terminal and the dashboard reflects it instantly.

🚪 How to open: SSH tunnel or nginx

By default, the dashboard goes up inlocalhost:8080inside the VPS. It is not exposed on the internet. You need to create asafe bridge— two common options:

🚇Option 1: SSH Tunnel (recommended for individual use)

ssh -L 8080:localhost:8080 user@vps.exemplo.com

Then openhttp://localhost:8080in your browser. Traffic travels encrypted via SSH. No one from the outside sees anything.

🌐Option 2: nginx reverse proxy + auth

For multiple users or fixed URL: configure nginx as a reverse proxy with HTTPS (Let's Encrypt) and basic authentication or Cloudflare Access.

location / {
  auth_basic "Hermes Dashboard";
  auth_basic_user_file /etc/nginx/.htpasswd;
  proxy_pass http://localhost:8080;
}

⚠️Don't do

Never open port 8080 directly on the internet without authentication. Admin token is not a firewall. Even if Hermes has login, you are exposing unnecessary attack surface.

TOPIC 3
3

⚡ Skill open-dashboard

Forgetting SSH command each time is unnecessary friction. The bundled skillabrir-dashboardautomates: you say it via Telegram"open my dashboard"and it runs the local tunnel, waits for the door to open, and gives you the link.

🔧 What the skill does

  1. Confirms that the VPS host is configured on~/.ssh/config
  2. Wheelssh -fN -L 8080:localhost:8080 vpsin background
  3. Wait for port 8080 to become available (timeout 10s)
  4. give you awayhttp://localhost:8080clickable
  5. Optionally opens the browser automatically (on Mac/Linux with GUI)

📌 Limitation

The skill assumes your SSH key and host are configuredlocally. It doesn't work if the agent is running inside the VPS — it would be trying to create a tunnel for itself. It makes sense in laptop setup → VPS.

TOPIC 4

🗒️ Kanban view

When you activate the v0.13.0 multi-agent system, the Kanban view becomes your dashboard. Classic layout (To Do, In Progress, Done) with extra layers to give workers visibility.

📋 To Do

Cards waiting for available worker. Each card shows title, description and retries count (if any).

⚙️ In Progress

Active cards with worker name, time since start, heartbeat indicator (green/yellow/red).

✅ Done

Completed. Hallucination gate green mark if passed, red if rejected and returned to the top.

🎯 Helpful visual cues

  • •Green Heartbeat= worker pinging in the last 30s
  • •Yellow Heartbeat= silent worker for 1-3 min, monitoring
  • •Red heartbeat / "zombie"= worker considered locked, task will be reclaimed
  • •Retry count visible= you see right away if a task is falling repeatedly
TOPIC 5

🔌 Plugin maintenance

The plugins tab on the dashboard is where you manage extensions:LCM(lossless compression),Kanban(multi-agent), calendar integrations, community plugins. Update becomes click instead of manual pull + restart.

🛠️ What do you manage there

  • •List:see all installed plugins, current version, available version
  • •To update:1 click to pull up new version; automatic restart of the plugin
  • •Disable:without uninstalling; useful for debugging ("is plugin X causing this?")
  • •Configure:edit plugin settings.yaml directly in the UI
  • •Logs:Plugin log stream for real-time debugging

⚠️Be careful with auto-update

Updating the plugin can break skills that depended on the old version. In production, read the changelog before upgrading. In development, update soon — you want to catch breaks early.

TOPIC 6

📵 When NOT to use a dashboard

Most daily usefits on Telegram. Dashboard is a maintenance tool, not an operation tool. Anyone who gets used to opening a dashboard for anything loses the practicality of the agent “in their pocket”.

✓ Telegram covers

  • ✓Asking for tasks, asking questions
  • ✓Check today's cron, upcoming executions
  • ✓Read daily/weekly summaries
  • ✓Adjust memory ("note that...")
  • ✓Add simple skill by description
  • ✓Pause/resume cron

✓ Dashboard wins when

  • ✓Edit long Markdown file (complex skill)
  • ✓Sort dozens of Kanban cards
  • ✓Search text in old sessions
  • ✓Update 5 plugins at once
  • ✓View token usage chart
  • ✓Configure new channels (Discord, Slack)

💡Rule of thumb

If you can order in text, order via Telegram. If you are editing a large file or sorting many visual items, open the dashboard.80/20 goes to Telegram.
