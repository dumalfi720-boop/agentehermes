🏛️ Origin and license

Hermes Agent is a project byNous Research(NousResearch), distributed under licenseMIT. It was released inJuly 22, 2025and in less than 1 year it became one of the most popular open source AI agents in the world.

📊Project numbers (May/2026)

  • 141,404stars on GitHub
  • 21,979forks
  • 7,938commits
  • v0.13.0"The Tenacity Release" (May/2026)
  • 88.7%Python, with TUI in TypeScript

⚖️What MIT means in practice

  • •You canuse commerciallywithout paying royalties
  • •He canmodify and redistributethe code
  • •He canembed in closed productwithout opening your
  • •You still pay for inference (API, cloud, electricity) — open source ≠ free to operate

📱 20+ messaging platforms

Hermes has anative gatewaythat connects the agent to messengers. It's not an external wrapper — it's a project module.hermes gateway setupRun a wizard.

🌐 Supported Platforms

Telegram, Discord, Slack, WhatsApp, Signal, SMS, Email, Home Assistant, Mattermost, Matrix, DingTalk, Feishu/Lark, WeCom, Weixin,BlueBubbles (iMessage), QQBot, Yuanbao, Microsoft Teams, Google Chat, and browser.

Each platform has a configurable toolset. Platforms like Signal (without message editing) have their own verbosity controls.

💡Where to start

TelegramThis is the recommended path for the first installation. BotFather delivers a token within 30 seconds. Discord requires developer portal. WhatsApp goes through an external gateway (Whapi, Meta Business). iMessage requires BlueBubbles on a Mac.

🔄 Self-improving by design

The big difference compared to a stateless chatbot: Hermespersists learningin three layers — memory (MEMORY.md, USER.md), skills (SKILL.md) and searchable history (SQLite with FTS5).

🧬 The 3 layers of learning

MEMORY.md (~800 tokens):agent's personal notes — environmental facts, decisions, projects. Self-managed via toolmemory.
USER.md (~500 tokens):your profile — preferences, style, personal rules.
SKILL.md:reusable procedures with YAML front matter, in~/.hermes/skills/.
SQLite + FTS5:Full session history, searchable bysession_search.

⚠️Attention

Self-improvingit's not magic. It only evolves if you correct it, save it, transform processes into skills and adjust the tone. Without user feedback, it becomes stagnant.

🆚 Hermes in the ecosystem

Hermesdoes not replaceClaude Code, OpenCode, Cursor or n8n. It occupies a specific niche:proactive agent via message, always on, with scheduled automations and self-improvement.

1

Claude Code = code cockpit

You sitting at the PC, inside the project, doing deep work. Claude model for reasoning. No message gateway.

2

Hermes = remote control and automations

You on the move, via Telegram. Schedule thing via cron. Recurring tasks that don't need you present.

How to combine

Maintain a GitHub repository with context, skills, and settings. Point Claude CodeeHermes for this repo. Each agent reads its own format (CLAUDE.md vs AGENTS.md vs .hermes.md). You work fluidly between the two.
