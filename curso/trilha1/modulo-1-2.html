MODULE 1.2

🧠 The mindset: ask Hermes himself

The biggest twist: you don't need to know everything before you start. The agent reads documentation, debugs itself, and proposes solutions — as long as you treat it as such.

📖 The agent reads the documentation itself

The official Hermes documentation is athttps://hermes-agent.nousresearch.com/docsand the agent himself knows how to consult it. When you ask him for something he doesn't know how to do, heYou can search the doc itself, in X/Twitter links, in GitHub READMEs, or in URLs you provide.

🪄 Phrases that unlock the agent

  • →"Research how to do X and tell me what you need from me."
  • →"Read this doc <link> and implement it in our environment."
  • →"I don't know how to configure Y. Look it up and give me step-by-step instructions."
  • →"This error appeared: <paste error>. Investigate and propose a solution."

💡Why does this reduce the curve

You don't need to memorize commands, cron syntax or skill structure. Ask and he thinks. Use the agent as a personal tutor for yourself.

2

🔗 Drop links and it implements

Paste a post into X, a GitHub repository or a product doc, and ask himread and try to implementthe idea or tool in your environment.

🧪 Pattern "vi demo → glue → test"

This flow radically accelerates experimentation. You convert content into your agent's real capacity in minutes:

  1. See someone demonstrating a new skill on Twitter
  2. Paste the link into Hermes
  3. Ask:"Evaluate if it makes sense for my setup. If so, install and test."
  4. He reads, plans and asks you the right questions along the way

⚠️Careful

Always review before approving permanent permissions. Not everything that appears in X is safe or applicable to your setup. Start with "once" permission before "always allow".

3

🐛 Errors become the agent’s own task

When something breaks (Telegram not responding, wrong .env, cron crashes), you paste the error or describe the symptom andask him to diagnose. It has tools to check processes, read logs and restart services.

✓ Good diagnostic prompt

"I sent a message on Telegram at 2:32 pm and the bot didn't respond. Other messages worked. Check the gateway, read gateway.log for the last 10 minutes and tell me what happened."

✗ Bad prompt

"It doesn't work."

No command, no error message, no context. The agent will spend 5 shifts just to understand what you mean.

4

🪜 Start small, always evolve

Don't try to set everything up on the first day. OneAgent MVPwell put together is worth more than an ambitious frankenstein that is difficult to debug.

🎯 Recommended MVP for the first 7 days

  1. VPS or Docker container on air
  2. Telegram connected and responding
  3. Private GitHub configured for backup
  4. A simple cron (daily midnight backup)
  5. Initial onboarding: you said who you are, projects, preferences

Just that. Use for a week. Then add skills and crons as life demands.

💡Principle

"First it works, then it gets better." Each new part tested in isolation is easier to maintain than 10 parts configured together on a Saturday afternoon.

5

👷 Treat like a new employee

Onboarding mindset: You don't give an intern full access on day 1. Same thing with Hermes — start withlimited scopes, expand as confidence increases.

✓ Good practices

  • ✓Tightly Scoped Tokens
  • ✓Separate accounts (AgentMail, dedicated Gmail)
  • ✓IPs locked when possible
  • ✓Revocable permissions in 1 click
  • ✓Agent-named keys

✗ What to avoid

  • ✗Token with full admin permission
  • ✗Your personal account logged into the agent
  • ✗Same key shared between agents
  • ✗"Always allow" for everything on day 1
  • ✗Credit card directly via API
6

📝 Training means continuing to use

Hermes is not finished configuring.Every mistake is an opportunityto upgrade memory or skill. Each success is a candidate to become a reusable process.

🔁 2 errors rule

  • →Wrongonce: correct and follow.
  • →Wrongtwice: stop, update skill or memory.
  • →Repeated instruction: becomes skill.
  • →Wrong tone: adjusts the soul.
  • →Asked for information he already had: saves in memory.

📈What changes in 30/60/90 days

  • Day 1-7:generic agent, still getting tone and details wrong
  • Day 30:memory stabilized, first useful skills, crons running
  • Day 60:aligned tone, your routine processes already automated
  • Day 90:truly proactive agent — reduces friction rather than creating
