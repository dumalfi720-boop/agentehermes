🖥️ Where to run Hermes

VPS, Docker, laptop, Mac Mini or Termux: each environment has trade-offs. Know how to choose before installing.

☁️ Cloud VPS

VPS is a computer rented in the cloud with a public IP, connected24/7. It is the default environment for reliable crons. Hostinger, Hetzner, DigitalOcean and Oracle (Always Free) are the options most used by the Hermes community.

2

🐳 Docker container

Docker isolates Hermes in a container, withyour own .env, your keys and your file system. The VPS is the building, the container is the room. It is the recommended path.

✓ Advantages of Docker

  • ✓Multiple agents on the same VPS without conflict
  • ✓Clean update without breaking the host
  • ✓Persistent volumes with easy backup
  • ✓Reproducible deployment via docker-compose

⚠️ Pranks

  • !.envfrom host ≠.envof the container
  • !Volume~/.hermes:/root/.hermesneeds to be assembled
  • !Container runs in UTC by default (TZ adjustment)
  • !Update viadocker pull, do not touch the live container

🔑 Key command

To save secrets inside the Docker container, use the Hermes command:

hermes config set GITHUB_TOKEN seu_token_aqui

This writes in/root/.hermes/.envinside the container, never in the conversation log.

3

💻 Personal Laptop

You install locally on Mac/Linux/WSL2. Works when the laptop isturned on, connected and unlocked. Good for development. Bad for serious crons — the agent "dies" every time you close the lid.

🎯 When to use a laptop

  • →Prototype and test skills before moving up to VPS
  • →Learn how to do initial onboarding without cloud costs
  • →Deep work via TUI/CLI when on PC

Migration to VPS is trivial: skills, memory and configs are just Markdown. Commit + clone + restart and that's it.

4

🍎 Mac Mini connected at home

A cheap Mac Mini (used M1/M2) connected 24/7 works as a home mini-VPS. You access via SSH orTailscaleand has good local performance for small models via Ollama.

✓ Strengths

  • • Operating cost ≈ zero (you already have energy at home)
  • • Run local Qwen or LLaMA without paying for cloud GPU
  • • No monthly hosting fee
  • • Low latency when you are at home

⚠️ Weak points

  • • Light drops break the agent
  • • Home internet is often unstable
  • • Dynamic DNS and home firewall require setup
  • • If you move, it takes weeks to normalize
5

📱 Android via Termux

Termux is a Linux terminal emulator for Android. You can run the Hermesliterally inside the cell phone, without cloud.

📱 Use cases

  • →Offline or on-the-go scenarios
  • →Extreme privacy (nothing in the cloud)
  • →Quick demos in presentation
  • →Environments without access to VPS (corporate network)

⚠️Real limits

Does not replace VPS for production. Battery, Android Doze mode, background process kill and resource limits affect crons. Use as a complement, not as a base.

6

🎯 How to decide — tree

It is not existential.Decide in 30 secondswith the tree below. 90% of cases end in VPS+Docker.

?

Do you need crons running 24/7 without you there?

Yes:VPS + Docker. He finished.

?

Do you just want to test and learn?

Yes:Laptop. Migrate later.

?

Extreme privacy? Local model? Leftover hardware?

Yes:Mac Mini or desktop with GPU. Tailscale for remote access.

?

No cloud or PC, just a cell phone?

Yes:Termux. But plan to move to VPS when you can.
