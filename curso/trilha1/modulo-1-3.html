VPS, Docker, laptop, Mac Mini ou Termux: cada ambiente tem trade-offs. Saiba escolher antes de instalar.

☁️ VPS na nuvem

VPS é um computador alugado na nuvem com IP público, ligado 24/7. É o ambiente padrão para crons confiáveis. Hostinger, Hetzner, DigitalOcean e Oracle (Always Free) são as opções mais usadas pela comunidade do Hermes.

💡Dica

Ubuntu 24.04 LTS é o padrão recomendado. Ative backup automático diário e scanner de malware grátis (na Hostinger é 1 clique). Configure firewall antes de subir secrets pro container.

2

🐳 Docker container

Docker isola o Hermes em um container, com seu próprio .env, suas chaves e seu sistema de arquivos. O VPS é o prédio, o container é a sala. É o caminho recomendado.

✓ Vantagens do Docker

  • ✓Múltiplos agentes na mesma VPS sem conflito
  • ✓Atualização limpa sem quebrar o host
  • ✓Volumes persistentes com fácil backup
  • ✓Deploy reproduzível via docker-compose

⚠️ Pegadinhas

  • !.env do host ≠ .env do container
  • !Volume ~/.hermes:/root/.hermes precisa estar montado
  • !Container roda em UTC por padrão (ajuste TZ)
  • !Atualização via docker pull, não mexer no container vivo

🔑 Comando-chave

Para salvar secrets dentro do container Docker, use o próprio comando do Hermes:

hermes config set GITHUB_TOKEN seu_token_aqui

Isso escreve em /root/.hermes/.env dentro do container, nunca no log da conversa.

3

💻 Laptop pessoal

Você instala localmente no Mac/Linux/WSL2. Funciona quando o laptop está ligado, conectado e desbloqueado. Bom pra desenvolvimento. Ruim pra crons sérios — o agente "morre" toda vez que você fecha a tampa.

🎯 Quando usar laptop

  • →Prototipar e testar skills antes de subir pra VPS
  • →Aprender a fazer onboarding inicial sem custo de cloud
  • →Trabalho profundo via TUI/CLI quando estiver no PC

Migração pra VPS é trivial: skills, memória e configs são só Markdown. Commit + clone + restart e pronto.

4

🍎 Mac Mini ligado em casa

Um Mac Mini barato (M1/M2 usado) ligado 24/7 funciona como mini-VPS doméstico. Você acessa via SSH ou Tailscale e tem performance local boa para modelos pequenos via Ollama.

✓ Pontos fortes

  • • Custo de operação ≈ zero (já tem energia em casa)
  • • Roda Qwen ou LLaMA local sem pagar cloud GPU
  • • Sem mensalidade de hosting
  • • Latência baixa quando você está em casa

⚠️ Pontos fracos

  • • Quedas de luz quebram o agente
  • • Internet residencial costuma ser instável
  • • Dynamic DNS e firewall doméstico exigem setup
  • • Se você se mudar, leva semanas pra normalizar
5

📱 Android via Termux

Termux é um emulador de terminal Linux pra Android. Dá pra rodar o Hermes literalmente dentro do celular, sem cloud.

📱 Casos de uso

  • →Cenários offline ou em viagem
  • →Privacidade extrema (nada na cloud)
  • →Demos rápidas em apresentação
  • →Ambientes sem acesso a VPS (rede corporativa)

⚠️Limites reais

Não substitui VPS pra produção. Bateria, Doze mode do Android, kill de processo em background e limites de recurso afetam crons. Use como complemento, não como base.

6

🎯 Como decidir — árvore

Não é existencial. Decida em 30 segundos com a árvore abaixo. 90% dos casos terminam em VPS+Docker.

?

Você precisa de crons rodando 24/7 sem você presente?

Sim: VPS + Docker. Acabou.

?

Você só quer testar e aprender?

Sim: Laptop. Migra depois.

?

Privacidade extrema? Modelo local? Hardware sobrando?

Sim: Mac Mini ou desktop com GPU. Tailscale pro acesso remoto.

?

Não tem cloud nem PC, só celular?

Sim: Termux. Mas planeje migrar pra VPS quando puder.
