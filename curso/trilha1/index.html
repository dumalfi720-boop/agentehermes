🪽 What is Hermes and why does it exist

Before configuring anything, understand the agent: origin, license, what comes installed, and how it positions itself in the AI ​​agent ecosystem.

What is it:

Hermes Agent is an open source project by Noose Research, distributed under the MIT license. It is one of the fastest growing open source AI agents on GitHub recently.

Why learn:

Understanding the origin helps to calibrate expectations: active community, fast release pace, no vendor lock-in. You can run it offline, modify it, sell it embedded in the product.

Key concepts:

Open source ≠ free to operate (you still pay for inference and hosting). MIT allows commercial use. Noose Research is the main maintainer, but the contribution is distributed.

What is it:

Hermes comes with 91 skills installed by default and access to a hub with more than 520 community skills, including 16 official Anthropic skills.

Why learn:

You don't start from scratch. Much of what you need for common tasks is already ready: GitHub auth, repo management, opening a dashboard, browser automation, image generation.

Key concepts:

Skill = reusable procedure in Markdown. Hub = public catalog. You install a skill by pasting the link and asking the agent to use it.

What is it:

The agent already understands images, automates the browser, generates images, speaks (TTS), executes commands in the terminal, plans tasks and responds with audio on Telegram.

Why learn:

Knowing what it already does avoids reinventing the wheel. You ask the task in natural language and it decides which tool to use. No need to code integration for the basics.

Key concepts:

Telegram voice notes are useful for long summaries. Browser automation works to fill out forms and scrape pages. TTS replaces reading when you are driving.

What is it:

Hermes can receive and respond on Telegram, Discord, Slack, WhatsApp or iMessage. Telegram is the recommended way to get started with BotFather's ease.

Why learn:

Messaging is what transforms the agent from a “tool in the terminal” to an “assistant always with me”. Without it, you only use it when you are at the computer.

Key concepts:

Each platform has its own friction. Telegram = bot token via BotFather. Discord = bot token via developer portal. WhatsApp = more complicated, usually via Whapi-type gateway.

What is it:

Hermes improves when learning is persisted as memory, skills, and searchable history. It's not magic — you have to correct, save processes and refine.

Why learn:

This is the difference compared to stateless chatbots. The more you use it, the more aligned it becomes. Repeated tasks become skills, preferences become memory, old sessions become searchable context.

Key concepts:

"Memory = what to remember. Skill = how to do it." Sessions are in SQLite. You need to understand: "save this in memory", "turn it into a skill", "don't repeat this error".

What is it:

Hermes does not replace Claude Code; they complement each other. Claude Code is the code cockpit (terminal + project). Hermes is the remote control (Telegram + crons). OpenCode falls between the two.

Why learn:

Choosing the wrong tool creates friction. Trying to use Hermes for hardcore coding is frustrating. Try using Claude Code to schedule tasks on your cell phone too.

Key concepts:

Use Claude Code sitting at the PC, with time. Use Hermes on the go, via Telegram. Maintain a central GitHub repo with context, skills and configs — any agent points to it.

🧠 The mindset: ask Hermes himself

The biggest twist: you don't need to know everything beforehand. The agent reads documentation, debugs itself, and proposes solutions — as long as you treat it as such.

What is it:

When you ask for something that he doesn't know how to do, he can search the documentation itself, X/Twitter links or URLs you provide.

Why learn:

Dramatically reduces the learning curve. You don't need to memorize commands, cron syntax or skill structure. Ask and he thinks.

Key concepts:

Magic phrase: "Research how to do X and tell me what you need from me." He reads, plans and asks you the right questions along the way.

What is it:

You paste a post link into X, from a repository or a doc, and ask it to read it and try to implement the idea or tool in your environment.

Why learn:

This “saw a cool demo → glue → test” flow accelerates experimentation. You convert content into your agent's real capacity in minutes.

Key concepts:

Always review before approving permanent permissions. Not everything that appears in X is safe or applicable to your setup.

What is it:

When something breaks (Telegram doesn't respond, wrong .env, cron fails), you paste the error or describe the symptom and ask him to diagnose it.

Why learn:

Don't become hostage to tutorials. Most of the common problems Hermes solves by checking processes, restarting services or consulting the doc.

Key concepts:

The better you describe the symptom (which command, which message, in which context), the better the diagnosis. "It doesn't work" is rarely enough.

What is it:

Don't try to set everything up on the first day. Upload the agent, connect Telegram, create a simple cron (backup) and use it for a few days before adding more.

Why learn:

Ambitious configuration all at once becomes a difficult-to-debug Frankenstein. Each new part tested in isolation is easier to maintain.

Key concepts:

Principle: "first it works, then it gets better." Agent MVP: Telegram + GitHub + 1 backup cron. Rest comes with real use.

What is it:

Onboarding mindset: You don't give an intern full access on day 1. Same thing with Hermes — start with limited scopes, expand as confidence grows.

Why learn:

Reduces blast radius if something goes wrong. Tightly scoped tokens, separate accounts, locked IPs, revocable permissions.

Key concepts:

Principle of least privilege. Each agent with its named key. Dedicated AgentMail or Gmail. Never your main account.

What is it:

The agent does not finish configuring. Each mistake is an opportunity to update memory or skill. Each success is a candidate to become a reusable process.

Why learn:

Those who treat Hermes as a static tool never make the most of it. Those who treat them as colleagues who learn will see the agent become more useful month by month.

Key concepts:

Rule of 2 mistakes: made a mistake twice → updates skill or memory. Repeated instruction → becomes skill. Wrong tone → adjusts soul.

🖥️ Where to run Hermes

VPS, Docker, laptop, Mac Mini or Termux: each environment has trade-offs. Know how to choose before installing.

What is it:

VPS is a computer rented in the cloud with a public IP, connected 24/7. Hostinger has Hermes auto-deploy in 1 click. Hetzner and DigitalOcean are cruder alternatives.

Why learn:

It is the default environment for reliable crons. Your machine can sleep, the VPS cannot. You access from anywhere via SSH or Telegram.

Key concepts:

Hostinger's KVM 2 (~$100/year) fits multiple agents. Ubuntu 24.04 LTS is the default. Daily automatic backup is practically mandatory.

What is it:

Docker isolates Hermes in a container, with its own .env, its own keys, and its own file system. The VPS is the building, the container is the room.

Why learn:

It is the recommended path. You run multiple agents (personnel, marketing, finance) without mixing keys or memory. Updates are cleaner.

Key concepts:

.env of the container ≠ .env of the VPS. Hermes reads the container. Command:hermes config set GITHUB_TOKEN xxxwrite in/opt/data/.env.

What is it:

You install locally on Mac/Linux. It works when the laptop is turned on, connected to the internet and unlocked.

Why learn:

Good for development, skills testing and initial exploration. Bad for serious crons — the agent "dies" every time you close the lid.

Key concepts:

Use your laptop to prototype. When the agent proves value, move to VPS. Skills, memory and configs are just Markdown — migration is commit + clone.

What is it:

A cheap Mac Mini (used M1/M2) connected 24/7 at home works as a mini-VPS. You access via SSH/Tailscale and have good local performance for small models.

Why learn:

Operating costs tend to be zero (you already have energy at home). Excellent for running Qwen or LLaMA locally without paying for cloud GPU.

Key concepts:

Pay attention to power outages, unstable home internet and dynamic DNS. Tailscale solves the network part without exposing public IP.

What is it:

Termux is a Linux terminal emulator for Android. You can literally run Hermes on your cell phone, without the cloud.

Why learn:

Offline scenarios, travel, low privacy required. Useful for quick demos or restricted environments without VPS access.

Key concepts:

Does not replace VPS for production. Battery, background kill by Android and resource limits are real.

What is it:

Important Crons? VPS+Docker. Just dev and testing? Laptop. Extreme privacy + local model? Mac Mini or desktop with GPU. Edge case offline? Termux.

Why learn:

90% of cases end in VPS+Docker. Knowing this avoids wasting days trying to make a personal laptop viable for things that need uptime.

Key concepts:

Start simple (laptop), validate the use, migrate to VPS when use becomes part of the routine. Migration is trivial: skills and memory are just Markdown versioned.

💼 Real use cases

Where Hermes really shines: people, marketing, finance, support, operations and content. Concrete examples of cron and skill by area.

What is it:

Daily summary of priorities at 8am, contextual reminders ("notify me of X tomorrow at 2pm"), quick searches, assisted reading of long texts.

Why learn:

It is the use case with the lowest risk and highest perceived return. The agent becomes useful in 1 day. You create a habit.

Key concepts:

Short crons (morning, end of day, Fridays). All via Telegram. No need for local GPU.

What is it:

Cron monitoring YouTube/Instagram comments, suggesting responses within the brand's tone, maintaining a live editorial calendar.

Why learn:

Marketing is where “proactive agent” saves the most time. You become a reviewer, not a creator from scratch.

Key concepts:

Soul.md becomes more important here — it sets the tone for the brand. Specific skills: "reply comment type X", "suggest thread about Y".

What is it:

Cron reading spreadsheets, bank or financial API, generating weekly summary, alerting about due dates and cost anomalies.

Why learn:

Replaces a lot of manual spreadsheets. Reduces "forgot to pay X" to zero. It fits perfectly into a separate agent according to the principle of least privilege.

Key concepts:

Financial agent must have different keys than the personal agent. Minimum permissions. Alerts via Telegram in a private channel.

What is it:

Hermes reading emails/tickets, classifying, suggesting answers based on FAQ or customer history, escalating whatever is critical.

Why learn:

High repeat message volume is the worst small operation bottleneck. The agent filters and leaves time for non-standard cases.

Key concepts:

Soul must be empathetic and professional. Memory must load FAQ. Agent never responds alone without review at the beginning.

What is it:

Weekly audit cron: open ports, suspicious processes, disk space, abnormal logs. Automatic daily backup.

Why learn:

You don't forget to keep the environment healthy. It's safer without having to become an SRE.

Key concepts:

Skill "security audit" is one of the most useful. Combine with external layer firewall (Hostinger Upguard or ufw).

What is it:

Skills to revise text in your style, generate variations for A/B, suggest topics based on past content, create a weekly newsletter.

Why learn:

Reduces "blank page" friction. Maintains voice consistency. Cron transforms "make newsletter" into "review the newsletter that the agent has already prepared".

Key concepts:

Soul.md = your voice. Memory.md carries examples of your good writing. Skill "review text" must preserve voice, not homogenize.
